Detailed Description

DomainCollector extends SerialDataCollector with domain management features. It allows to efficiently read subdomains (sub-partitions) from multi-process hdf5 files with entries annotated with domain information.

DataContainer * DCollector::DomainCollector::readDomain ( int32_t  id,

Dimensions  domainOffset,

DomDataClass *  dataClass,
bool  lazyLoad = false 
) throw (DCException) [virtual]

Efficiently reads domain-annotated data. In comparison to the standard read method, only data required for the requested partition is read. It passes back a DataContainer holding all subdomains or subdomain portions, respectively, which make up the requested partition. Data from the files is immediately copied to the DomainData objects in the container and DomainData buffers are dealloated when the DomainData object is deleted, which must be done by the user.

Template Parameters:
DomDataTypedata type of the data to be read (e.g. int, float, ...)
Parameters:
idid of the group to read from
namename of the dataset
domainOffsetdomain offset for reading (logical start of the requested partition)
domainSizedomain size for reading (logical size of the requested partition)
dataClassoptional domain type annotation (can be NULL)
lazyLoadset to load only size information for each subdomain, data must be loaded later using readDomainLazy
Returns:
returns a pointer to a newly allocated DataContainer holding all subdomains
